# Privacy Policy

**App:** Capsule for Android (`dev.withcapsule.android`)\
**Effective date:** 20 June 2026\
**Last updated:** 20 June 2026

Capsule is a file-transfer app built to be privacy-friendly by design. This
policy explains exactly what information the app does and does not handle, where
it goes, and the choices you have. We have deliberately kept data collection to
the minimum needed to run the service and understand, in aggregate, how the app
is used.

If you self-host the Capsule server, all data stays entirely within infrastructure that you control.

## Summary

- **No accounts.** Capsule does not ask you to register, log in, or provide a
  name, email address, or phone number.
- **No advertising and no ad identifiers.** We do not use advertising SDKs and
  do not collect the Android Advertising ID.
- **No third-party tracking or marketing cookies.**
- **Analytics are anonymous, self-hosted, and optional.** Usage analytics are
  processed by our own Umami instance and can be turned off at any time in
  **Settings**.
- **Your files and your decryption keys are never used in analytics.** File
  contents, file names, and the encryption passphrase / 12-word mnemonic never
  leave your device through analytics, and the mnemonic is never transmitted
  anywhere by any Capsule services.

## 1. Information we do **not** collect

- We do not collect your name, email address, phone number, or any account
  credentials (as there are no accounts).
- We do not collect your contacts, calendar, microphone audio, or your precise
  GPS location.
- We do not use the Android Advertising ID or any cross-app/advertising
  tracking.
- We do not sell or rent any data to anyone.

## 2. Files you transfer

When you upload a file, the app sends that file to a Capsule server, with the options being:

- the **default hosted server** at `https://send.withcapsule.dev`, or
- a **custom / self-hosted server** that you configure in Settings.

When a user downloads a file, the app retrieves it from the configured server
using the file ID or link the user provides.

The handling of files on the server (storage location, retention, expiry) is
governed by the operator of that server. If you use the default hosted server,
that operator is the Capsule project; if you self-host, that operator is **you**.

**Encryption.** If you enable encryption, the file is encrypted on your device
_before_ it is uploaded, and the server only ever receives encrypted blob
data. The encryption passphrase / 12-word mnemonic is generated locally, shown on
your device, and is **never transmitted by the app** anywhere. Not to the server and not
to analytics. If you lose the mnemonic, an encrypted file cannot be recovered.

## 3. Anonymous usage analytics

To understand how the app is used and to improve it, Capsule records anonymous,
aggregate usage events using **Umami**, an open-source, privacy-focused
analytics tool that we self-host. As a result, analytics data is not shared with any
third-party analytics provider.

Analytics are enabled by default but can be switched off at any time via
**Settings → Anonymous analytics**. When disabled, no analytics events are sent except 
for a final "opted out" signal.

### What analytics events contain

Each event records the in-app screen path (for example `/upload`, `/download`,
`/settings`), an action name (for example `app_launched`, `upload_success`,
`download_error`), and your device's screen resolution.

Once per session, the app also records two non-identifying device attributes so
we can understand what hardware the app runs on:

- device model (for example "samsung SM-S928U1"),
- Android version (for example "Android 16").

Your approximate region/country is also derived on our server from the network
connection your request arrives on. Consistent with how Umami works, **your IP
address is not stored**; it is used only momentarily to derive an approximate
region and is then immediately and completely discarded. Umami does not use
tracking cookies, and unique-visitor counts are based on a daily-rotating, salted
hash rather than a persistent identifier. There is no stable identifier that ties
events back to you across days.

### Non-identifying attributes on some events

Some analytics events include additional non-identifying attributes that
describe _what_ happened, never _who_ you are. Examples:

- whether a transfer was encrypted (`true` / `false`),
- whether an action succeeded or failed,
- the selected theme, server option, or scanner target.

Two categories deserve specific mention so you have the full picture:

- **File identifiers** - A few events (such as viewing a transfer's status,
  deleting a file from the server, or removing it from your local history)
  include the **file ID**. This is a random identifier generated by the server
  for a transfer. It is not derived from you or your device and is not linked to
  your identity, but it does relate to a specific transfer.
- **Error messages** - Upload and download error events include the technical
  error message so we can diagnose failures. These messages describe the error
  (for example a network or server status condition) and may, in uncommon cases,
  contain a technical detail such as a server address you configured.

File contents, file names, and decryption keys are never sent nor used in analytics.

## 4. Camera (QR code scanning)

The app requests camera access only so a user can scan QR codes containing a
file ID, a download link, or a decryption mnemonic. QR scanning is performed
on-device using Google's ML Kit barcode scanner. The camera preview is
processed locally to read the code; images are not stored by the app and are not
uploaded anywhere. You can decline the camera permission and still use the app by
entering IDs, links, and mnemonics manually. It is merely a convenience, not a necessity.

## 5. Data stored on your device

The app stores the following locally on-device:

- **Transfer history** - file metadata such as file IDs and basic transfer
  details. History does not include decryption mnemonics.
- **Settings** - your preferences such as the selected server, download folder,
  theme, haptics, onboarding state, and the analytics opt-out.

This data stays on your device. You can clear transfer history from within the
app, and uninstalling the app removes this local data.

## 6. How we use information

- **Files** are used solely to perform the transfer you request.
- **Anonymous analytics** are used only in aggregate to understand feature usage,
  measure reliability (for example success vs. error rates), and prioritize
  improvements.

We do not use any of this information for advertising, profiling, or automated
decision-making, and we do not sell it.

## 7. Third parties

Capsule does not integrate third-party advertising or marketing services. The
components that process data are:

- **Capsule server** (default or custom) - receives the files you transfer.
- **Self-hosted Umami** - receives anonymous analytics events, unless opted out. 
  This is operated by the Capsule project, not a third-party analytics company.
- **Google ML Kit** - runs on-device to decode QR codes; it does not transmit anything

## 8. Data retention

- **Analytics** - retained only in aggregate for understanding usage trends.
- **Transferred files** - retained according to the policy of the server operator
  (the Capsule project for the default server, or you if you self-host).
- **On-device data** - retained until user clears history or uninstalls the app.

## 9. Your choices and rights

- **Turn analytics off** at any time in Settings
- **Decline camera access** and use manual entry instead.
- **Use a self-hosted server** so that transferred files stay within
  infrastructure you control.
- **Clear local data** by clearing history in the app or uninstalling it.

Because analytics are anonymous and not tied to an identity, we generally cannot
locate or single out data relating to a specific individual. Depending on your
jurisdiction (for example under the GDPR), you may have rights to access,
correct, or erase personal data; if you believe you have provided us with
personal data, contact us using the details in Section 13.

## 10. Children's privacy

Capsule is not directed to children and does not knowingly collect personal
information from children.

## 11. International transfers

The default server is run in the United States. Custom servers may have data processing in other countries.
Capsule does not control those and has no jurisdiction over them.
